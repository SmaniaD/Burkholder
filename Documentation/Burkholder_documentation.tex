\section{Overview}

This repository proves the {\it Burkholder inequality for the real-valued
martingale transforms} in the Lean proof assistant, as stated in \noindent and in  \texttt{MartingaleTransforms.lean} and Figure 1.
\begin{figure}[h]
  \includegraphics[width=0.8\textwidth]{maintheorem.png}
  \caption{}
\end{figure}

That is the {\bf Burkholder inequality for the real-valued
martingale transforms  } 

\begin{theorem}[Burkholder martingale transform inequality]
\label{thm:Lp_Burkholder_inequality_martingaleTransform}
Let $p\in (1,\infty)$. Let \((\Omega,\mathcal{A})\) be a
measurable space, and let \(\mu\) be a finite measure on \(\Omega\).  Let
$\F=(\F_n)_{n\in \mathbb{N}}$ be a filtration indexed by \(\N\), and let
\[
  w_n,f_n :  \Omega \to \R.
\]
be measurable functions  such that $f =(f_n)_{n\in \mathbb{N}}$ is a discrete martingale with respect to  $\F$. Moreover
$$
  f_n \in L^p(\mu)
  \qquad \text{for every } n\in\N.
$$
 Suppose that   $w=(w_n)_{n\in \mathbb{N}}$  predictable sequence with respect to $\F$ with 
$$
  |w_n(\omega)| \le 1
  \qquad \text{for every } n\in\N
  \text{ and for } \mu\text{-a.e. } \omega.
$$
Then, for every $n\in\N$,
$$
  \norm{(w\star_m f)_n}_{L^p(\mu)}
    \le
  \bigl(p^\ast-1\bigr)\norm{f_n}_{L^p(\mu)},
$$
where $w\star_m f$ is the martingale transform of $f$ with respect to $w$ and  $p^\ast =\max \{ p, (p-1)/p\}.$
\end{theorem}

See  Ba{\~n}uelos and  Davis 2011 \cite{BaBu} for more information on this interesting result.


\section{The Burkholder Function \(v\)}

The proof follows Burkholder 1985 \cite{zbMATH03903326}. The main step there and the  analytic heart of the formalization is to prove
that the function 
$$
  v(p,x,y)
    =
      \left|\frac{x+y}{2}\right|^p
      -
      |p^\ast-1|^p
      \left|\frac{x-y}{2}\right|^p.
$$
has a majorant $u$ satisfying the theorem in Figure 2 and as stated in  \texttt{Majorant.lean}.  That is, 

\begin{figure}[t]
 \includegraphics[width=0.8\textwidth]{majorant.png}
 \caption{}
\end{figure}

\begin{theorem}
Let \(p\in\mathbb{R}\) with \(p>1\).  Then there exist   \[
  u,\ \frac{\partial u}{\partial x},\ \frac{\partial u}{\partial y}
    :  \mathbb{R}\to\mathbb{R}
\]
and a constant $C\geq 0$ such that 
\begin{itemize}
\item[A.]  The functions
\[
  (x,y)\mapsto u(x,y),\qquad
  (x,y)\mapsto \frac{\partial u}{\partial x}(x,y),\qquad
  (x,y)\mapsto \frac{\partial u}{\partial y}(x,y)
\]
are continuous on \(\mathbb{R}^2\), 
\item[B.]  For all \(x,y,h,k\in\mathbb{R}\),
\[
  |u(x,y)| \le C\bigl(|x|^p+|y|^p\bigr),
\]
\[
  \left|\frac{\partial u}{\partial x}(x,y)\right|
    \le C\bigl(|x|^{p-1}+|y|^{p-1}\bigr),
\]
\[
  \left|\frac{\partial u}{\partial y}(x,y)\right|
    \le C\bigl(|x|^{p-1}+|y|^{p-1}\bigr),
\]
\item[C.] We have
\[
  hk\le 0
  \Longrightarrow
  u(x+h,y+k)
    \le
  u(x,y)
    + \frac{\partial u}{\partial x}(x,y)h
    + \frac{\partial u}{\partial y}(x,y)k,
\]
\item[D.]  We have 
\[
  v(p,x,y)\le u(x,y),
\]
\item[E.]  We have
\[
  xy\le 0 \Longrightarrow u(x,y)\le 0,
\]
and
\item[F.] We have
\[
  p\ne 2,\ xy=0,\ (x,y)\ne(0,0)
  \Longrightarrow
  u(x,y)<0.
\]
\end{itemize} 
\end{theorem}

The proof of Theorem 2 is particularly painful to formalize in Lean. The function $u$ is given explicitly in \cite{zbMATH03903326}, but it is defined piecewise on several sectors of $\mathbb{R}^2$. Although the argument used to prove Theorem 2 in \cite{zbMATH03903326} takes only about half a page, is fairly simple, and could be followed by an undergraduate student with a solid background in point-set topology and analysis, a fully  formalization would probably require several weeks of work (as Codex itself suggested to me),  and perhaps even months. 

In our case, the formalization became feasible within a manageable amount of time, namely a few days, by using AI tools such as Copilot and Codex in {\it agent mode.} In practice, we barely had to type any Lean code ourselves; instead, we mostly directed the agent when needed. Unfortunately, even with the paid version of Codex — through ChatGPT Plus, together with the Codex extension for Visual Studio Code — we frequently ran out of credits every day, and had to spend time purchasing additional credits and restarting sessions in different computing environments.

The result was more than 20,000 lines of Lean code. We believe that this code is unlikely to be optimal; nevertheless, it is remarkable that we barely had to write any proof by hand. Most of the work consisted in guiding the agent using colloquial language and image snapshots from \cite{zbMATH03903326}. This is quite astonishing when compared with the capabilities of pre-agent AI tools only a few months ago.
